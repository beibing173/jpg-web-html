📌 静态版本说明

由于GitHub Pages只支持静态文件托管，无法运行后端服务，因此此版本提供了预制的演示文件来展示HTML伪装技术的效果。 在实际应用中，您需要部署带有后端服务的完整版本才能实现动态的文件上传和URL生成功能。

🔧 技术实现

静态版本实现方式：

  • 文件重命名：将HTML文件重命名为.jpg扩展名
  • MIME类型：Web服务器根据文件内容而非扩展名确定MIME类型
  • 浏览器解析：浏览器检测到HTML内容后自动渲染为网页
  • GitHub Pages：利用GitHub Pages的静态文件托管能力

完整版本功能：

  • 动态上传：支持用户实时上传HTML文件
  • URL生成：自动为每个文件生成唯一的伪装URL
  • 访问统计：记录每个伪装URL的访问次数
  • 内容处理：自动处理Base64编码和普通HTML内容
